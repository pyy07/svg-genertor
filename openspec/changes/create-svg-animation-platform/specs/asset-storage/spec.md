## ADDED Requirements

### Requirement: 素材保存
系统 SHALL 保存所有生成的 SVG 动画素材。

#### Scenario: 自动保存生成的素材
- **WHEN** 用户成功生成 SVG 动画
- **THEN** 系统自动将 SVG 代码保存到数据库
- **AND** 保存生成时间戳
- **AND** 保存用户输入的自然语言描述
- **AND** 如果用户已登录，关联用户 ID

#### Scenario: 保存素材元数据
- **WHEN** 素材被保存
- **THEN** 系统保存以下元数据：
  - 生成时间
  - 用户输入描述
  - 用户 ID（如果已登录）
  - SVG 代码
  - 密码 ID（用于追踪）

### Requirement: 素材查看
系统 SHALL 允许用户查看已保存的素材。

#### Scenario: 查看所有素材
- **WHEN** 用户访问素材列表页面
- **THEN** 系统显示所有已保存的素材
- **AND** 显示素材的预览图和基本信息
- **AND** 按生成时间倒序排列

#### Scenario: 查看个人素材
- **WHEN** 已登录用户访问个人素材库
- **THEN** 系统仅显示该用户生成的素材
- **AND** 显示素材的预览图和详细信息

#### Scenario: 查看素材详情
- **WHEN** 用户点击查看素材详情
- **THEN** 系统显示完整的 SVG 动画
- **AND** 显示生成时间和原始描述
- **AND** 显示 SVG 源代码（可选）

### Requirement: 素材持久化
系统 SHALL 确保素材数据持久化存储，不会丢失。

#### Scenario: 数据库存储
- **WHEN** 素材被保存
- **THEN** 数据被写入数据库
- **AND** 使用事务确保数据一致性
- **AND** 失败时返回错误信息

#### Scenario: 数据备份
- **WHEN** 系统部署到生产环境
- **THEN** 数据库配置自动备份
- **AND** 备份策略确保数据可恢复
